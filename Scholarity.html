1. Abstract

BACHsql este o platforma ce permite manipularea mai multor baze de date, doar in condiitile in care utilizatorul curent are un cont inregistrat pe aplicatie. Acesta, de asemenea, putand colabora si cu alti utilizatori existenti. Platforma este contruita in intregime pe standarde deschise. Acest document reprezentand atat, modul in care aplicatia prezinta diferite functionalitati, cat si modul in care este codificata.

2. Introduction

Industria IT este intr-o continua dezvoltare pe piata curenta. Majoritatea aplicatiilor ce doresc o posibila interactiune intre utilizatori sau retinerea acestora pentru o mai buna comunicare cu clientii, indiferent de scopul dorit, aplicatia necesita o baza de date solida si bine integrata. Platforma BACHsql vine in ajutorul dezvoltatorilor cu o interfata pe cat se poate de atragatoare si usor de folosit.

3. Structure

Structura platformei este bazata pe modelul bine cunoscut, MVC(Model-view-controller), ce este un model arhitectural. Succesul modelului se datoreaza izolarii logicii de bussines fata de considerentele interfetei cu utilizatorul, rezultand o aplicatie unde aspectul vizual sau/ si nivelele inferioare ale regulilor de bussines sunt mai usor de modificat, fara a afecta alte nivele. Pe langa MVC, elementul controlor(API) este accesat prin AJAX, prescurtarea pentru Asynchronous. Fiind o tehnica de programare pentru crearea de aplicatii web interactive rapide. O structurare in profunzime a aplicatiei sunt prezentate in sectiunile urmatoare.

Micro Services

  • 1. Account Service

    • Microserviciul pentru conturi a

  • 2. DataBase Functions

    • Cel de-al doilea micro serviciu se preocupa de manipularea bazelor de date.Atat crearea acestora, adaugare de noi tabele/ functii/ procedurii sau queries, cat si modificarea sau stergerea bazelor de date nedorite.

  • 3. Collaborate

    • Acest micro serviciu reda posibilitatea utilizatorilor sa interactioneze. Invite-ul unui utilizator se va face prin intermediul e-mailului. Doi sau mai multi utilizatori vor putea comunica si sa aduca posibile modificari intr-o maniera LIVE.

  • 4. Generate Code

    • Ultimul micro serviciu se va preocupa de partea de generare a codului, codul MySQL se va transpune intr-unul dintre limbajale de programare disponibile si anume Java/ PHP.
